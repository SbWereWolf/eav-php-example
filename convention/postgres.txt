Правила для Postgres

Правила именования
Имя должно быть содержательным.
Не следует включать тип данных в имя.
Слова в имени должны быть в единственном числе.
Префиксы и суффиксы недопускаются.
Имя не должно совпадать с зарезервированным словом СУБД.
Имена колонок не должны совпадать с именами таблиц.
Для именования следует использовать только слова английского языка.
Сокращения и абревиатуры не допускаются, использование вместо букв других символов недопускается.
Имена записываются snake_case.
Имя должно содержать минимум одно существительное, желательно что бы к существительному прилагалось поясняющее прилагательное или глагол.
Имена колонок для флагов должны содержать глагол : "is" / "has" / "was".
Примари кей всегда называется "id", тип всегда INTEGER.
Шаблон для имени первичного ключа : pk_<имя_содержащей_таблицы>_<колонка_индекса>.
Шаблон для имени индекса : ix_<имя_содержащей_таблицы>_<колонка_индекса>.
Шаблон для имени уникального индекса : ux_<имя_содержащей_таблицы>_<колонка_индекса>.
Шаблон для именования внешних ключей : fk_<имя_содержащей_таблицы>_<имя_таблицы_с_индексом>_<колонка_индекса>.
Шаблон для именования поля внешнего ключа : <имя_таблицы_с_индексом>_<колонка_индекса>.
Шаблон для именования таблицы стыковки двух таблиц <имя_одной_таблицы>_<имя_другой_таблицы>.

Типы данных для колонок
Колонки в соответствии с их назначением должны иметь следующие типы :
 - числовые индексы - INTEGER
 - строковые индексы - char(20, 50, 100, 200)
 - числа - DOUBLE PRECISION
 - строки до 4000 символов - varchar(4000)
 - строки длинее 4000 - text
 - отметки времени timestamptz
 - интервалы времени interval

Правила оформления запросов
Ключевые слова следует писать в верхнем регистре.
Функции SQL и СУБД следует писать в верхнем регистре.
Имена объектов СУБД не надо брать в ковычки.
Следует избегать использовать функции СУБД и другие ключевые слова, если существуют аналогичные функции ANSI SQL.
Комментарии следует писать с использованием блочного комментирования ( /* */ )
При объявлении альясов следует в явном виде использовать AS
Ключевые слова определяющие части запроса, такие как : SELECT, FROM, WHERE, GROUP и другие следует писать в отдельной строке.
