Правила работы с Git

1
Отдельный функционал в отдельной ветке
2
Слияние веток выполнять либо для синхронизации кода с кодом других веток, либо по завершению работы над веткой
3
Камиты выполнять по завершению работы над веткой, либо для синхронизации кода с другими ветками.
Допускается выполнить один комит в конце рабочего дня.
4
Перед слиянием ветки функционала ( feature/ ) в ветку разработки ( develop/ ),
 следует слить ветку разработки в ветку функионала,
 исправить конфликты в ветке функционала,
 закомитить исправления в ветку функционала и только после этого
  ветку функционал сливать с веткой разработки.
